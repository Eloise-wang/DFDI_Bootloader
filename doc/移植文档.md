## 1 概述

本文档描述将 S32K142 CAN UDS Bootloader 核移植到杰发 AC7840 系列车规级 MCU 的完整技术方案。

### 1.1 源芯片 vs 目标芯片规格对比

| 维度 | S32K142 (源) | AC7840 (目标) | 影响 |
|---|---|---|---|
| 内核 | ARM Cortex-M4F @ 80MHz | ARM Cortex-M4F @ 120MHz | 向上兼容 |
| 内核架构 | Armv7E-M + FPU | Armv7E-M + FPU | 完全一致 |
| PFlash 总容量 | 256KB (单区) | 1MB PFlash (双 Bank 各 512KB) | 容量充裕，支持双区 OTA |
| DFlash | 无独立 DFlash (FlexNVM) | 128KB 独立 DFlash | 新增存储区用于 APP 信息 |
| SRAM | 32KB + 4KB FlexRAM | 124KB + 4KB FlexRAM | RAM 交换区地址需重新规划 |
| Flash 擦除粒度 | 4KB 扇区 | ~2KB 页 | 擦除循环次数增加 |
| Flash 擦写时间 | 批量擦除 ~20ms | 批量擦除 ~28ms，页擦除 ~820us | 喂狗周期需延长 |
| Flash ECC | 支持 | PFlash/DFlash 均支持 ECC | 需要处理 ECC 错误标志 |
| CAN 通道 | 3 个 FlexCAN (1 个支持 FD) | 4 个 CAN-FD | 兼容，通道数更多 |
| 安全引擎 | CSEc (AES-128 + Secure Boot) | CSE (AES-128 + SHE + TRNG) | 接口不同，需重写 Seed/Key |
| 看门狗 | WDOG/EWDOG | WDOG/EWDOG | 兼容 |
| DMA | 16 通道 | 16 通道 DMA | 兼容 |
| 功能安全 | ISO 26262 ASIL-B | ISO 26262 ASIL-B | 一致 |
| 开发工具 | S32 Design Studio 3.4 + GCC | 杰发官方 SDK + IAR/GCC | 需要重新配置工具链 |
| OTA 方案 | 单区 Flash | 双 Bank Flash + 硬件 SWAP | 原有框架可复用，OTA 方案升级 |

## 3 BSP 层 — 完全重写

BSP (Board Support Package) 层包含所有与芯片强相关的代码，S32K142 的 BSP 代码全部需要重写。

### 3.1 main.c 重写要点

S32K142 的 `main.c` 中 BSP 初始化部分需要完全重写为 AC7840 版本。核心接口对照如下：

#### (1) 初始化函数对照

| S32K142 (原) | AC7840 (目标) |
|---|---|
| `CAN_Init(&can_pal1_instance, &can_pal1_Config0)` | `CAN_Init(CAN0, &can_config)` |
| `CAN_InstallEventCallback(&can_pal1_instance, ...)` | `CAN_InstallCallback(CAN0, ...)` |
| `CAN_ConfigRxBuff(&can_pal1_instance, ...)` | `CAN_SetRxBuffer(CAN0, ...)` |
| `CAN_ConfigTxBuff(&can_pal1_instance, ...)` | `CAN_SetTxBuffer(CAN0, ...)` |
| `CAN_Send(&can_pal1_instance, ...)` | `CAN_Transmit(CAN0, ...)` |
| `CAN_Receive(&can_pal1_instance, ...)` | `CAN_Receive(CAN0, ...)` |
| `INT_SYS_EnableIRQ(CAN0_ORed_0_15_MB_IRQn)` | `NVIC_EnableIRQ(CAN0_IRQn)` |
| `CLOCK_SYS_Init(...)` | AC7840 时钟初始化 |
| `PINS_DRV_Init(...)` | AC7840 引脚初始化 |
| `InitFlash()` | AC7840 Flash 初始化 |

#### (2) CAN 接收回调适配

原 `CAN_RXTXMsgHandle()` 在接收到 CAN 帧时将数据写入 TP 层。AC7840 适配版本：

```c
// AC7840 BSP/main.c

void CAN0_IRQHandler(void)
{
    uint32 id;
    uint8 data[64];
    uint8 len;

    // 读取 AC7840 CAN Rx FIFO 中的数据
    if (CAN_ReceiveFrame(CAN0, &id, &len, data) == OK)
    {
        // 写入 TP 层 — 与原项目逻辑完全一致
        TP_DriverWriteDataInTP(id, len, data);
    }

    // 重新使能接收
    CAN_Receive(CAN0, ...);
}

// CAN 发送完成回调
void CAN_TxCompleteCallback(void)
{
    TP_DoTxMsgSuccesfulCallback();
}
```

#### (3) 主循环发送函数适配

原 `SendMsgMainFun()` 从 TX FIFO 读数据并发送。适配版本：

```c
// AC7840 BSP/main.c

static void SendMsgMainFun_AC7840(void)
{
    uint8 aucMsgBuf[64];
    uint32 msgId = 0;
    uint32 msgLength = 0;

    if (TP_DriverReadDataFromTP(64, &aucMsgBuf[0], &msgId, &msgLength) == TRUE)
    {
        // 调用 AC7840 的 CAN 发送接口
        CAN_Transmit(CAN0, msgId, msgLength, aucMsgBuf);
    }
}
```

### 3.2 CAN 配置参数

S32K142 当前配置的波特率计算（基于 40MHz 时钟）：

```
波特率 = 40MHz / (preDivider+1) / (propSeg+phaseSeg1+phaseSeg2+3) / (rJumpwidth+1)
       = 40MHz / 6 / 13 / 2
       = 512 kbps
```

AC7840 波特率需要根据其外设时钟重新计算。关键配置项对照：

| 配置项 | S32K142 当前值 | AC7840 配置建议 |
|---|---|---|
| CAN 通道 | CAN1 | 选择一个通道 (如 CAN0) |
| 波特率 | 512 kbps | 保持 500kbps 或 1Mbps |
| 采样点 | ~87.5% | 保持相近 |
| 是否有 CAN FD | `enableFD = true` (接收) | 根据需求配置 |
| Mailbox/Rx buffer 数量 | 7 个 | 查 AC7840 手册 |
| 中断类型 | ORed MB IRQ | 查 AC7840 中断方案 |

### 3.3 时钟初始化

AC7840 时钟树与 S32K142 不同，需要在 `clock_config.c` 中重新配置：

- 配置系统时钟源 (PLL / 外部晶振 / 内部 RC)
- 配置 CAN 外设挂载在哪条时钟线上
- 配置 Flash 控制器工作频率 (AC7840 要求 8~120MHz)

### 3.4 引脚复用

在 `pin_mux.c` 中配置：

| 功能 | S32K142 引脚 | AC7840 引脚 (需查手册) |
|---|---|---|
| CAN0_Tx | PTB0 / PTB1 等 | 查 AC7840 datasheet |
| CAN0_Rx | PTB1 / PTB0 等 | 查 AC7840 datasheet |
| SWD_DIO | - | 保留调试口 |
| SWD_CLK | - | 保留调试口 |

## 4 HAL 层 — 需要适配

### 4.1 Flash HAL (🔴 高优先级，需重写)

Flash HAL 是移植工作量最大、最关键的部分。

#### 4.1.1 架构差异分析

| 对比项 | S32K142 | AC7840 |
|---|---|---|
| 架构 | 单区 PFlash + FlexNVM | **双 Bank PFlash + 独立 DFlash** |
| 擦除粒度 | 4KB 扇区 | ~2KB 页 |
| 编程粒度 | 8 字节对齐 | 64 位 (8 字节) 对齐 |
| ECC | 支持，地址映射不同 | PFlash/DFlash 均支持 |
| 擦除 API | `FLASH_DRV_EraseSector()` | 查 AC7840 SDK `FLS_Init()` 等 |
| 编程 API | `FLASH_DRV_Program()` | 查 AC7840 SDK `FLS_Program()` 等 |

#### 4.1.2 AC7840 内存布局建议

```
AC7840 PFlash 内存映射 (建议):

0x00000000 ─┬─ [Bootloader]   0x00000000 ~ 0x0000FFFF   (64KB)
            │
0x00010000 ─┤─ [APP_A]        0x00010000 ~ 0x0007FFFF   (448KB, Bank0 剩余)
Bank0       │
            │
0x00080000 ─┤─ [APP_B]        0x00080000 ~ 0x000FFFFF   (512KB, Bank1 全部)
Bank1       │
            │
0x00100000 ─┴─ [Bank1 End]

AC7840 DFlash / FlexRAM:
0x10000000 ─┬─ [DFlash]       0x10000000 ~ 0x1001FFFF   (128KB, 存 APP 信息/指纹)
            │
0x20000000 ─┤─ [SRAM]         0x20000000 ~ 0x2002EFFF   (~124KB)
FlexRAM     │
0x200F0000 ─┤─ [FlexRAM]      0x200F0000 ~ 0x200F0FFF   (4KB, 可用作 EEPROM 仿真)
```

#### 4.1.3 flash_hal_Cfg.c 适配

需要修改以下配置项：

```c
// ===== AC7840/AC7840_CAN/public_inc/user_config.h =====

// 更换 MCU 类型
#define MCU_TYPE (MCU_AC7840_TYPE)

// 定义 AC7840 MCU 类型
#define MCU_AC7840_TYPE (5)

// ===== AC7840/HAL/flash_hal/flash_hal_Cfg.c =====

// 修改扇区大小
#if (MCU_TYPE == MCU_AC7840_TYPE)
#define SECTOR_LEN (2048u)         // AC7840 页擦除 ~2KB
#define MAX_ERASE_SECTOR_FLASH_MS (40u)  // 页擦除最大时间，留余量
#endif

// 修改 APP 可用空间
#ifdef EN_SUPPORT_APP_B
// APP_A: Bank0 中 Bootloader 之后的空间
const BlockInfo_t gs_astBlockNumA[] = {
    {0x00010000u, 0x00080000u},   // 0x10000 ~ 0x80000, 448KB
};

// APP_B: Bank1 全部空间
const BlockInfo_t gs_astBlockNumB[] = {
    {0x00080000u, 0x00100000u},   // 0x80000 ~ 0x100000, 512KB
};
#else
// 单区模式
const BlockInfo_t gs_astBlockNumA[] = {
    {0x00010000u, 0x00100000u},   // 0x10000 ~ 0x100000, 960KB
};
#endif

// 修改 APP 信息存储地址 (存放在 DFlash 中)
const BlockInfo_t gs_astFlashDriverBlock[] = {
    {0x10000000u, 0x10020000u},   // DFlash 128KB 区域
};

// 修改向量表偏移量
#define APP_VECTOR_TABLE_OFFSET (0x200u)
#define RESET_HANDLE_OFFSET (4u)
#define RESET_HANDLER_ADDR_LEN (4u)
```

#### 4.1.4 driver/flash/flash.c 重写

S32K142 使用 `flash.c` 中的 `FLASH_DRV_EraseSector()` 和 `FLASH_DRV_Program()` 调用 NXP SDK Flash 驱动。AC7840 需要调用杰发 SDK 的 Flash 驱动：

```c
// AC7840/driver/flash/flash.c

#include "ac7840_flash.h"  // 杰发 SDK Flash 头文件

static flash_config_t s_flashConfig;

void InitFlash(void)
{
    // AC7840 Flash 初始化
    FLS_Init(&s_flashConfig);
}

void InitFlashAPI(void)
{
    // AC7840 不需要 Flash Driver 加载，直接初始化即可
    // 如果 AC7840 SDK 也支持将 Flash 驱动拷贝到 RAM 运行，逻辑同 S32K142
}

uint8 EraseFlashSector(uint32 logicalAddr, uint32 eraseLen)
{
    status_t ret;

    // AC7840 擦除：页擦除或批量擦除
    ret = FLS_EraseSector(&s_flashConfig, logicalAddr, eraseLen);
    return (ret == STATUS_SUCCESS) ? 0 : 1;
}

uint8 WriteFlash(uint32 startAddr, const void *dataBuf, uint16 dataLen)
{
    status_t ret;

    // AC7840 编程：64-bit 对齐编程
    ret = FLS_Program(&s_flashConfig, startAddr, dataLen, dataBuf);
    return (ret == STATUS_SUCCESS) ? 0 : 1;
}
```

### 4.2 CRC HAL (🟡 中优先级)

S32K142 当前使用软件 CRC (`EN_CRC_SOFTWARE` 启用)。如果需要使用 AC7840 硬件 CRC：

| 操作 | S32K142 | AC7840 |
|---|---|---|
| 软件 CRC | `crc_hal.c` 中手写算法 | **可直接复用** |
| 硬件 CRC | `CRC_DRV` NXP SDK | AC7840 内置 CRC 模块，寄存器不同 |

**建议**：直接使用软件 CRC，完全避免硬件 CRC 寄存器差异，代码零改动。

### 4.3 Timer HAL (🟡 中优先级)

定时器 HAL 提供 1ms 和 100ms 软件定时器基准。S32K142 使用 LPTMR (低功耗定时器)。AC7840 有 `TIMER` 单元 (6 个 8 通道 PWM 单元中的 1 个通道可作为定时器)。

核心适配点：替换定时器中断初始化和中断处理函数，`timer_hal.c` 中的定时器管理逻辑可复用。

### 4.4 Watchdog HAL (🟢 低优先级)

```c
// watchdog_hal.c — 喂狗逻辑完全相同，只需替换底层寄存器访问
#if (MCU_TYPE == MCU_AC7840_TYPE)
#define WATCHDOG_FEED()  FSL_WDOG_Refresh(WDOG)
#else
#define WATCHDOG_FEED()  WDOG_DRV_Refresh()
#endif
```

### 4.5 UDS_algorithm_hal / AES (🟡 中优先级)

这是移植中**最复杂的 HAL 模块之一**。S32K142 使用 CSEc 模块 (通过 NXP SDK 与安全引擎交互)，AC7840 使用 CSE 模块 (支持 SHE 规范)。

核心差异：

| 对比项 | S32K142 (CSEc) | AC7840 (CSE) |
|---|---|---|
| 密钥存储 | 内置密钥槽 | **17 个用户密钥槽** |
| Seed/Key 协议 | 通过 Mailbox 交互 | 通过 Mailbox 交互 |
| API 接口 | NXP SDK `SEC_DRV_*` | 杰发 SDK `CSE_*` |

`UDS_alg_hal.c` 中的 Seed/Key 验证流程需要重写以适配 AC7840 CSE 的 Mailbox 接口。具体步骤：

1. 调用 `CSE_Init()` 初始化 CSE 模块
2. UDS 0x27 请求 Seed：调用 `CSE_GenerateRandom()` 获取随机数作为 Seed
3. UDS 0x27 发送 Key：调用 `CSE_VerifyKey()` 验证解密后的 Key
4. 替换 `AES.c` 中的 AES 实现为 AC7840 CSE 的 AES-128 接口

| `flash_hal_Cfg.c` | `gs_astBlockNumA` / `gs_astBlockNumB` 分区定义 |
| `fls_app.c` | `Flash_GetNewestAPPType()` / `Flash_GetOldAPPType()` A/B 区切换逻辑 |
| `boot_Cfg.c` | `Boot_RemapApplication()` 自动选择最新版本跳转 |
| `uds_app_cfg.c` | UDS 下载时自动校验目标区合法性 |

### 5.2 AC7840 启用双区

#### 5.2.1 开启宏

```c
// user_config.h
#define EN_SUPPORT_APP_B
```

#### 5.2.2 分区地址配置

在 `flash_hal_Cfg.c` 中配置正确的 Bank 地址（参见 4.1.3 节）。

#### 5.2.3 AC7840 硬件 SWAP (可选高级特性)

AC7840 支持**硬件 SWAP** 功能，可通过编程 Flash 信息区实现 Bank 地址重映射：

| SWAP 状态 | 启动行为 |
|---|---|
| SWAP A (默认) | Bank0 映射到 0x00000000，CPU 从 Bank0 启动 |
| SWAP B | Bank1 映射到 0x00000000，CPU 从 Bank1 启动 |

配置方式：向 Flash 信息区写入 SWAP 标志位，然后复位即可切换启动 Bank。

**推荐初期使用软切换方式**（设置跳转标志位 + 复位），后续再引入硬件 SWAP。

## 9 已知风险与规避

| 风险 | 描述 | 规避措施 |
|---|---|---|
| Flash 操作期间 CPU 挂死 | AC7840 Flash 操作对总线频率敏感 | 确保 Flash 总线频率在 8~120MHz 范围内，操作期间关闭中断 |
| ECC 错误误报 | AC7840 Flash 全程 ECC，首次烧录可能有 ECC 初始化 | Bootloader 首次启动时初始化 ECC 状态 |
| 看门狗超时 | 擦除大扇区时间较长 | 在 Flash 擦写循环中定期喂狗 |
| CAN 中断丢失 | AC7840 CAN 接收 buffer 深度有限 | 确保及时读取 Rx FIFO，避免溢出 |
| Seed/Key 验证失败 | CSE 与 CSEc 的 Mailbox 协议细节不同 | 仔细对比两个芯片的安全引擎手册中 Mailbox 操作时序 |
| 双 Bank 切换失效 | SWAP 配置写入位置错误 | 先使用软切换方式（设置标志位 + 复位），确认逻辑正确后再使用硬件 SWAP |
