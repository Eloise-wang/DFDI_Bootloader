# RCM_SRS 复位原因整理

## 说明

`RCM_SRS` 是复位源状态寄存器，不同 bit 表示不同复位原因。

本工程中的定义位置：

- `DFDI_Bootloader\CMSIS\Driver\inc\rcm_drv.h`

其中和当前问题最相关的是以下几个 bit：

- `BIT17 = 0x00020000`：`LVR_RST`，低压复位
- `BIT18 = 0x00040000`：`SW_RST`，软件复位
- `BIT19 = 0x00080000`：`LOCKUP_RST`，内核锁死复位
- `BIT20 = 0x00100000`：`ACK_ERR_RST`
- `BIT21 = 0x00200000`：`WDG_RST`，看门狗复位
- `BIT26 = 0x04000000`：`EXT_RST`，外部复位引脚复位

## 常见值

### 1. `0x00040000`

含义：

- `SW_RST`

说明：

- 纯软件复位
- 常见于 `NVIC_SystemReset()` 或软件主动触发复位

### 2. `0x00200000`

含义：

- `WDG_RST`

说明：

- 纯看门狗复位
- 下载过程中如果出现这个值，基本可以判断为看门狗超时

### 3. `0x00240000`

计算：

- `0x00240000 = 0x00200000 + 0x00040000`

含义：

- `WDG_RST + SW_RST`

说明：

- 看门狗复位，同时带有软件复位标志
- 在 APP 请求进入 bootloader 的流程里，比较容易看到这个组合值

### 4. `0x00060000`

计算：

- `0x00060000 = 0x00040000 + 0x00020000`

含义：

- `SW_RST + LVR_RST`

说明：

- 软件复位
- 同时记录到了低压复位标志

### 5. `0x04070000`

计算：

- `0x04070000 = 0x04000000 + 0x00040000 + 0x00020000 + 0x00010000`

当前按工程已知定义可明确识别：

- `EXT_RST + SW_RST + LVR_RST`

说明：

- 外部复位引脚复位
- 软件复位标志存在
- 低压复位标志存在
- 其余未明确定义的 bit 需要结合芯片手册进一步确认

## 当前项目里的使用建议

### 看门狗问题排查

如果 `RCM_SRS` 中包含：

- `0x00200000`

优先判断为：

- 看门狗复位

例如：

- `0x00200000`
- `0x00240000`

这两个值都应优先按“看门狗相关复位”处理。

### 软件复位判断

如果 `RCM_SRS` 中包含：

- `0x00040000`

说明流程中发生过软件复位。

例如：

- `0x00040000`
- `0x00240000`
- `0x00060000`

### 电源/电压问题判断

如果 `RCM_SRS` 中包含：

- `0x00020000`

说明存在低压复位标志，需要同时关注：

- 供电是否稳定
- 下载瞬间电流变化
- 外设启动时是否拉低电压

## 结合当前问题的结论

对于目前 bootloader / APP 下载问题：

- `0x00200000`：说明下载过程中发生了看门狗复位
- `0x00240000`：说明既有看门狗复位，也有软件复位标志

因此当前排查重点应放在：

- 下载期间是否应临时关闭看门狗
- A/B 区擦写时 CPU 是否被 Flash 操作长时间阻塞
- 看门狗恢复时机是否合适
