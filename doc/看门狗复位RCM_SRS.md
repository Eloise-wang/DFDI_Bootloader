# RCM_SRS 复位原因整理

## 说明

`RCM_SRS` 是复位源状态寄存器，不同 bit 表示不同复位原因。

本工程中的定义位置：

- `DFDI_Bootloader\CMSIS\Driver\inc\rcm_drv.h`

## Bit 对照表

| Bit | 十六进制掩码 | 宏定义 | 含义 |
| --- | --- | --- | --- |
| BIT17 | `0x00020000` | `LVR_RST` | 低压复位 |
| BIT18 | `0x00040000` | `SW_RST` | 软件复位 |
| BIT19 | `0x00080000` | `LOCKUP_RST` | 内核锁死复位 |
| BIT20 | `0x00100000` | `ACK_ERR_RST` | ACK 错误复位 |
| BIT21 | `0x00200000` | `WDG_RST` | 看门狗复位 |
| BIT26 | `0x04000000` | `EXT_RST` | 外部复位引脚复位 |

## 常见值对照表

| `RCM_SRS` 值 | 组成 | 含义 | 说明 |
| --- | --- | --- | --- |
| `0x00040000` | `0x00040000` | `SW_RST` | 纯软件复位，常见于 `NVIC_SystemReset()` 或软件主动复位 |
| `0x00200000` | `0x00200000` | `WDG_RST` | 纯看门狗复位，下载过程中出现时基本可判断为看门狗超时 |
| `0x00240000` | `0x00200000 + 0x00040000` | `WDG_RST + SW_RST` | 看门狗复位，同时带有软件复位标志 |
| `0x00060000` | `0x00040000 + 0x00020000` | `SW_RST + LVR_RST` | 软件复位，同时记录到了低压复位标志 |
| `0x04070000` | `0x04000000 + 0x00040000 + 0x00020000 + 0x00010000` | `EXT_RST + SW_RST + LVR_RST` | 外部复位标志、软件复位标志、低压复位标志同时存在；其余未明确定义 bit 需结合芯片手册确认 |

## 排查建议表

| 判断条件 | 优先结论 | 建议排查方向 |
| --- | --- | --- |
| 包含 `0x00200000` | 看门狗复位 | 下载期间是否应临时关闭看门狗；擦写 Flash 时 CPU 是否被长时间阻塞；看门狗恢复时机是否合适 |
| 包含 `0x00040000` | 发生过软件复位 | 是否执行过 `NVIC_SystemReset()`、软件复位接口或复位流程 |
| 包含 `0x00020000` | 存在低压复位标志 | 供电是否稳定；下载瞬间电流变化；外设启动时是否拉低电压 |
| 包含 `0x04000000` | 外部复位引脚触发 | 复位引脚是否被外部电路拉低；是否有调试器/下载器干预 |

## 结合当前项目

| 当前值 | 结论 | 当前应重点关注 |
| --- | --- | --- |
| `0x00200000` | 下载过程中发生了看门狗复位 | 下载阶段是否关闭看门狗；A/B 区擦写是否导致 CPU 卡死 |
| `0x00240000` | 同时存在看门狗复位和软件复位标志 | 看门狗超时是否发生在软件复位流程附近 |
